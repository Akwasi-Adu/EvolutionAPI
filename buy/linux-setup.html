1 Prerequisites

Operating System

Ubuntu 20.04+, Debian 11+, RHEL 8+, or similar

.NET Runtime

.NET 9.0 ASP.NET Core Runtime

SQL Server Access

Network access to Sage Evolution SQL Server

Reverse Proxy

Nginx (recommended) or Apache

Note: EvoConnect connects to SQL Server where Sage Evolution data resides. SQL Server can be on a Windows machine on your network; the API itself runs on Linux.

2 Install .NET 9.0 Runtime

Ubuntu 22.04 / 24.04

Terminal
# Add Microsoft package repository wget https://packages.microsoft.com/config/ubuntu/$(lsb_release -rs)/packages-microsoft-prod.deb -O packages-microsoft-prod.deb sudo dpkg -i packages-microsoft-prod.deb rm packages-microsoft-prod.deb # Install ASP.NET Core Runtime sudo apt-get update sudo apt-get install -y aspnetcore-runtime-9.0

Debian 11 / 12

Terminal
# Add Microsoft package repository wget https://packages.microsoft.com/config/debian/$(cat /etc/debian_version | cut -d. -f1)/packages-microsoft-prod.deb -O packages-microsoft-prod.deb sudo dpkg -i packages-microsoft-prod.deb rm packages-microsoft-prod.deb # Install ASP.NET Core Runtime sudo apt-get update sudo apt-get install -y aspnetcore-runtime-9.0

RHEL / CentOS / Fedora

Terminal
# Add Microsoft repository sudo rpm -Uvh https://packages.microsoft.com/config/rhel/8/packages-microsoft-prod.rpm # Install ASP.NET Core Runtime sudo dnf install aspnetcore-runtime-9.0

Verify Installation

dotnet --list-runtimes # Expected output includes: # Microsoft.AspNetCore.App 9.0.x [/usr/share/dotnet/shared/Microsoft.AspNetCore.App] # Microsoft.NETCore.App 9.0.x [/usr/share/dotnet/shared/Microsoft.NETCore.App]

3 Deploy API Files

Create Application Directory

Terminal
# Create directory structure sudo mkdir -p /var/www/evoconnect sudo mkdir -p /var/www/evoconnect/logs # Create a dedicated user for the service (optional but recommended) sudo useradd -r -s /sbin/nologin evoconnect

Extract and Copy Files

Terminal
# Upload Published.zip to server (using scp, sftp, etc.) # scp Published.zip user@yourserver:~/ # Extract the ZIP file unzip Published.zip -d ~/evoconnect-temp # Copy contents to application directory sudo cp -r ~/evoconnect-temp/net9.0/* /var/www/evoconnect/ # Clean up rm -rf ~/evoconnect-temp ~/Published.zip # Set ownership sudo chown -R evoconnect:evoconnect /var/www/evoconnect # Set permissions sudo chmod -R 755 /var/www/evoconnect sudo chmod -R 775 /var/www/evoconnect/logs

4 Configure Application Settings

Edit the configuration file:

Terminal
sudo nano /var/www/evoconnect/appsettings.json

Key Configuration Sections

appsettings.json
{ "Logging": { "LogLevel": { "Default": "Information", "Microsoft.AspNetCore": "Warning" } }, "AllowedHosts": "*", // CORS - Add your application domains "AllowedOrigins": [ "https://yourapplication.com" ], // JWT Configuration - CHANGE THESE! "Jwt": { "Key": "your-super-secret-jwt-key-minimum-32-characters", "Issuer": "EvoConnect", "Audience": "EvoConnectClients", "ExpirationInMinutes": 60 }, // API Keys "ApiKeys": [ "your-secure-api-key-here" ], // SQL Server connection (Windows SQL Server on network) "Companies": { "company1": { "DisplayName": "My Company Ltd", "EvolutionDatabase": "Server=192.168.1.100;Database=Evolution_Company;User Id=sa;Password=YourPassword;TrustServerCertificate=True;", "EvolutionCommon": "Server=192.168.1.100;Database=SageCommon;User Id=sa;Password=YourPassword;TrustServerCertificate=True;" } }, "Evolution": { "DefaultCompany": "company1" }, "Features": { "EnableSwagger": true, "EnableDetailedErrors": false } }
SQL Server Connection from Linux: Use SQL Server Authentication (User Id/Password) when connecting from Linux. Windows Authentication (Trusted_Connection) typically doesn't work from Linux without complex Kerberos setup.

SQL Server Network Configuration

Ensure SQL Server allows remote connections:

  1. SQL Server must be configured to allow TCP/IP connections
  2. SQL Server port (default 1433) must be accessible from the Linux server
  3. Windows Firewall on SQL Server must allow inbound connections on port 1433

Test connectivity from Linux:

# Install SQL Server tools (optional, for testing) curl https://packages.microsoft.com/keys/microsoft.asc | sudo apt-key add - curl https://packages.microsoft.com/config/ubuntu/$(lsb_release -rs)/prod.list | sudo tee /etc/apt/sources.list.d/msprod.list sudo apt-get update sudo apt-get install mssql-tools unixodbc-dev # Test connection /opt/mssql-tools/bin/sqlcmd -S 192.168.1.100 -U sa -P 'YourPassword' -Q "SELECT @@VERSION"

5 Create systemd Service

Create a systemd service to run EvoConnect as a daemon that starts automatically on boot.

Terminal
sudo nano /etc/systemd/system/evoconnect.service

Add the following content:

evoconnect.service
[Unit] Description=EvoConnect - Sage Evolution REST API After=network.target [Service] Type=notify User=evoconnect Group=evoconnect WorkingDirectory=/var/www/evoconnect ExecStart=/usr/bin/dotnet /var/www/evoconnect/EvolutionSDKAPIsimp.dll Restart=always RestartSec=10 KillSignal=SIGINT SyslogIdentifier=evoconnect # Environment variables Environment=ASPNETCORE_ENVIRONMENT=Production Environment=ASPNETCORE_URLS=http://localhost:5000 Environment=DOTNET_PRINT_TELEMETRY_MESSAGE=false # Security hardening (optional) NoNewPrivileges=true PrivateTmp=true [Install] WantedBy=multi-user.target

Enable and Start the Service

Terminal
# Reload systemd to recognize new service sudo systemctl daemon-reload # Enable service to start on boot sudo systemctl enable evoconnect # Start the service sudo systemctl start evoconnect # Check status sudo systemctl status evoconnect

Useful Service Commands

# Stop the service sudo systemctl stop evoconnect # Restart the service sudo systemctl restart evoconnect # View logs sudo journalctl -u evoconnect -f # View last 100 log entries sudo journalctl -u evoconnect -n 100
Test locally: At this point, the API should be running on http://localhost:5000. Test with: curl http://localhost:5000/api/v1/license/machine-id

6 Configure Nginx (Reverse Proxy)

Nginx acts as a reverse proxy, handling HTTPS termination and forwarding requests to Kestrel.

Install Nginx

Terminal
# Ubuntu/Debian sudo apt-get install nginx # RHEL/CentOS sudo dnf install nginx

Create Nginx Configuration

Terminal
sudo nano /etc/nginx/sites-available/evoconnect
evoconnect (nginx config)
server { listen 80; server_name api.yourdomain.com; # Change to your domain or IP location / { proxy_pass http://localhost:5000; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection keep-alive; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto $scheme; proxy_cache_bypass $http_upgrade; # Timeout settings for long-running requests proxy_connect_timeout 60s; proxy_send_timeout 60s; proxy_read_timeout 60s; } }

Enable the Site

Terminal
# Create symbolic link to enable site sudo ln -s /etc/nginx/sites-available/evoconnect /etc/nginx/sites-enabled/ # Test nginx configuration sudo nginx -t # Reload nginx sudo systemctl reload nginx

Optional: Enable HTTPS with Let's Encrypt

Terminal
# Install Certbot sudo apt-get install certbot python3-certbot-nginx # Obtain SSL certificate (follow prompts) sudo certbot --nginx -d api.yourdomain.com # Auto-renewal is set up automatically # Test renewal with: sudo certbot renew --dry-run

Firewall Configuration

Terminal
# UFW (Ubuntu/Debian) sudo ufw allow 'Nginx Full' sudo ufw allow ssh sudo ufw enable # Or allow specific ports sudo ufw allow 80/tcp sudo ufw allow 443/tcp

7 Test the Installation

Test Endpoints

Terminal
# Test via Nginx (external) curl http://api.yourdomain.com/api/v1/license/machine-id # Or using IP address curl http://YOUR_SERVER_IP/api/v1/license/machine-id # Test Swagger UI # Open in browser: http://api.yourdomain.com/swagger

Expected Response

{ "machineId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx", "status": "awaiting_activation" }
Success! Copy your Machine ID and proceed to request your license.

Activate Your License

After receiving your license.lic file:

# Copy license file to application directory sudo cp license.lic /var/www/evoconnect/ sudo chown evoconnect:evoconnect /var/www/evoconnect/license.lic # Option 1: Reload license via API curl -X POST http://localhost:5000/api/v1/license/reload # Option 2: Restart the service sudo systemctl restart evoconnect

8 Troubleshooting

Service Won't Start

# Check service status sudo systemctl status evoconnect # View detailed logs sudo journalctl -u evoconnect -n 100 --no-pager # Try running manually to see errors cd /var/www/evoconnect sudo -u evoconnect dotnet EvolutionSDKAPIsimp.dll

Permission Denied Errors

# Fix ownership sudo chown -R evoconnect:evoconnect /var/www/evoconnect # Fix permissions sudo chmod -R 755 /var/www/evoconnect sudo chmod -R 775 /var/www/evoconnect/logs

Cannot Connect to SQL Server

  • Verify SQL Server allows TCP/IP connections
  • Check Windows Firewall on SQL Server (allow port 1433)
  • Test network connectivity: telnet 192.168.1.100 1433
  • Ensure SQL Server Authentication is enabled (not just Windows Auth)
  • Check connection string format in appsettings.json

502 Bad Gateway (Nginx)

  • Check if EvoConnect service is running: sudo systemctl status evoconnect
  • Verify the app is listening on port 5000: curl http://localhost:5000/api/v1/health
  • Check Nginx error logs: sudo tail -f /var/log/nginx/error.log

Application Logs

# View systemd journal logs sudo journalctl -u evoconnect -f # If file logging is enabled, check logs directory ls -la /var/www/evoconnect/logs/
