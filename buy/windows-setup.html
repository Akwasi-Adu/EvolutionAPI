1 Prerequisites

Operating System

Windows Server 2016+ or Windows 10/11

IIS Version

IIS 10.0 or later

SQL Server Access

Network access to Sage Evolution database server

.NET Runtime

.NET 9.0 Hosting Bundle

Enable IIS (if not already enabled)

If IIS is not installed, enable it via PowerShell (run as Administrator):

PowerShell (Admin)
# For Windows Server Install-WindowsFeature -Name Web-Server -IncludeManagementTools # For Windows 10/11 Enable-WindowsOptionalFeature -Online -FeatureName IIS-WebServerRole, IIS-WebServer, IIS-CommonHttpFeatures, IIS-HttpErrors, IIS-ApplicationDevelopment, IIS-NetFxExtensibility45, IIS-ASPNET45

Or enable via Control Panel ‚Üí Programs ‚Üí Turn Windows features on or off ‚Üí Internet Information Services.

2 Install .NET 9.0 Hosting Bundle

The ASP.NET Core Hosting Bundle includes the .NET Runtime and IIS support. This is required for running ASP.NET Core apps on IIS.

Download Link:
https://dotnet.microsoft.com/en-us/download/dotnet/9.0

Look for "Hosting Bundle" under the Windows section (not just the Runtime).

Installation Steps:

  1. Download the ASP.NET Core 9.0 Hosting Bundle installer
  2. Run the installer as Administrator
  3. Follow the installation wizard
  4. Important: Restart IIS after installation:
    Command Prompt (Admin)
    net stop was /y net start w3svc

Verify Installation:

Command Prompt
dotnet --list-runtimes # You should see something like: # Microsoft.AspNetCore.App 9.0.x [C:\Program Files\dotnet\shared\Microsoft.AspNetCore.App] # Microsoft.NETCore.App 9.0.x [C:\Program Files\dotnet\shared\Microsoft.NETCore.App]

3 Extract API Files

Create a directory for the API and extract the downloaded ZIP file.

Recommended Location:

C:\inetpub\wwwroot\EvoConnect\

Steps:

  1. Create the folder: C:\inetpub\wwwroot\EvoConnect
  2. Extract Published.zip contents
  3. Copy the contents of the net9.0 folder to C:\inetpub\wwwroot\EvoConnect\

Your folder structure should look like:

C:\inetpub\wwwroot\EvoConnect\ ‚îú‚îÄ‚îÄ appsettings.json ‚îú‚îÄ‚îÄ appsettings.Development.json ‚îú‚îÄ‚îÄ EvolutionSDKAPIsimp.dll ‚îú‚îÄ‚îÄ EvolutionSDKAPIsimp.exe ‚îú‚îÄ‚îÄ EvolutionSDKAPIsimp.xml ‚îú‚îÄ‚îÄ web.config ‚Üê Create this file (see below) ‚îú‚îÄ‚îÄ license.lic ‚Üê Add after receiving from support ‚îî‚îÄ‚îÄ [other DLLs and folders...]

Create web.config File:

Create a web.config file in the EvoConnect folder with this content:

web.config
<?xml version="1.0" encoding="utf-8"?> <configuration> <location path="." inheritInChildApplications="false"> <system.webServer> <handlers> <add name="aspNetCore" path="*" verb="*" modules="AspNetCoreModuleV2" resourceType="Unspecified" /> </handlers> <aspNetCore processPath="dotnet" arguments=".\EvolutionSDKAPIsimp.dll" stdoutLogEnabled="true" stdoutLogFile=".\logs\stdout" hostingModel="InProcess"> <environmentVariables> <environmentVariable name="ASPNETCORE_ENVIRONMENT" value="Production" /> </environmentVariables> </aspNetCore> </system.webServer> </location> </configuration>
Note: Create a logs folder in the EvoConnect directory for stdout logging. This helps with troubleshooting startup issues.

4 Configure IIS

4.1 Create Application Pool

  1. Open IIS Manager (run inetmgr)
  2. Right-click Application Pools ‚Üí Add Application Pool...
  3. Configure:
    • Name: EvoConnectPool
    • .NET CLR version: No Managed Code
    • Managed pipeline mode: Integrated
  4. Click OK
Critical: Set .NET CLR version to "No Managed Code". ASP.NET Core runs its own .NET runtime and does not use the legacy .NET Framework CLR.

4.2 Configure Application Pool Identity (for SQL Server)

If your SQL Server uses Windows Authentication (Trusted Connection), configure the App Pool identity:

  1. Right-click EvoConnectPool ‚Üí Advanced Settings...
  2. Under Process Model ‚Üí Identity
  3. Click ... and select:
    • Custom account: Enter a domain account with SQL Server access
    • Or use ApplicationPoolIdentity and grant SQL access to IIS APPPOOL\EvoConnectPool

4.3 Create Website or Application

Option A: Create as a new Website (recommended for dedicated server)

  1. Right-click Sites ‚Üí Add Website...
  2. Configure:
    • Site name: EvoConnect
    • Application pool: EvoConnectPool
    • Physical path: C:\inetpub\wwwroot\EvoConnect
    • Binding: HTTP, Port 5000 (or your preferred port)

Option B: Add as Application under Default Web Site

  1. Right-click Default Web Site ‚Üí Add Application...
  2. Configure:
    • Alias: evoconnect
    • Application pool: EvoConnectPool
    • Physical path: C:\inetpub\wwwroot\EvoConnect
  3. Access at: http://localhost/evoconnect

4.4 Set Folder Permissions

Grant the Application Pool identity permission to the folder:

  1. Right-click C:\inetpub\wwwroot\EvoConnect ‚Üí Properties
  2. Go to Security tab ‚Üí Edit ‚Üí Add
  3. Enter: IIS APPPOOL\EvoConnectPool
  4. Grant Read & Execute permissions
  5. For the logs folder, also grant Write permissions

5 Configure Application Settings

Edit appsettings.json to configure your Sage Evolution database connections and security settings.

appsettings.json
{ "Logging": { "LogLevel": { "Default": "Information", "Microsoft.AspNetCore": "Warning" } }, "AllowedHosts": "*", // CORS - Add your application domains "AllowedOrigins": [ "https://yourapplication.com", "http://localhost:3000" ], // JWT Configuration - CHANGE THESE VALUES! "Jwt": { "Key": "your-super-secret-jwt-key-minimum-32-characters-long", "Issuer": "EvoConnect", "Audience": "EvoConnectClients", "ExpirationInMinutes": 60 }, // API Keys for authentication "ApiKeys": [ "your-secure-api-key-here" ], // Multi-Company Database Configuration "Companies": { "company1": { "DisplayName": "My Company Ltd", "EvolutionDatabase": "Server=SQLSERVER;Database=Evolution_Company;Trusted_Connection=True;TrustServerCertificate=True;", "EvolutionCommon": "Server=SQLSERVER;Database=SageCommon;Trusted_Connection=True;TrustServerCertificate=True;" } }, // Default company to use "Evolution": { "DefaultCompany": "company1", "DefaultTimeout": 30, "EnableTracing": false, "MaxRetryAttempts": 3 }, // Features "Features": { "EnableSwagger": true, "EnableDetailedErrors": false } }
Security Warning:
  • Change the default JWT Key to a unique, secure value (minimum 32 characters)
  • Replace API keys with secure, randomly generated keys
  • Set EnableDetailedErrors to false in production
  • Restrict AllowedOrigins to only your application domains

Connection String Options

Windows Authentication (Recommended):

"Server=SQLSERVER;Database=Evolution_DB;Trusted_Connection=True;TrustServerCertificate=True;"

SQL Server Authentication:

"Server=SQLSERVER;Database=Evolution_DB;User Id=sa;Password=YourPassword;TrustServerCertificate=True;"

6 Test the Installation

6.1 Start the Website

  1. In IIS Manager, select your site
  2. Click Start in the Actions panel (if not already running)

6.2 Test the API

Open your browser and navigate to:

# Swagger UI (if enabled) http://localhost:5000/swagger # Health check endpoint http://localhost:5000/api/v1/health # Get Machine ID for license activation http://localhost:5000/api/v1/license/machine-id
Success! If you see the Swagger UI or a JSON response, your installation is working. Continue to the Getting Started guide to activate your license.

6.3 Activate Your License

After receiving your license.lic file:

  1. Copy license.lic to C:\inetpub\wwwroot\EvoConnect\
  2. Call the reload endpoint or restart the Application Pool:
    # Option 1: Reload via API curl -X POST http://localhost:5000/api/v1/license/reload # Option 2: Restart App Pool via PowerShell Restart-WebAppPool -Name "EvoConnectPool"

7 Troubleshooting

HTTP Error 500.19 - Invalid Configuration

  • Ensure the ASP.NET Core Hosting Bundle is installed
  • Restart IIS: net stop was /y && net start w3svc
  • Check web.config syntax

HTTP Error 502.5 - Process Failure

  • Check the logs\stdout files for detailed errors
  • Verify .NET 9.0 runtime is installed: dotnet --list-runtimes
  • Try running the app directly: dotnet EvolutionSDKAPIsimp.dll

HTTP Error 500.30 - In-Process Handler Load Failure

  • Ensure Application Pool is set to "No Managed Code"
  • Check that all required DLLs are present
  • Verify hostingModel="InProcess" in web.config

Cannot Connect to SQL Server

  • Verify SQL Server is accessible from the web server
  • Check connection string in appsettings.json
  • If using Windows Auth, ensure App Pool identity has SQL access
  • Test connection using SQL Server Management Studio from the web server

CORS Errors

  • Add your application domain to AllowedOrigins in appsettings.json
  • Restart the Application Pool after changes

View Detailed Logs

Enable stdout logging in web.config:

stdoutLogEnabled="true" stdoutLogFile=".\logs\stdout"

Then check C:\inetpub\wwwroot\EvoConnect\logs\ for log files.
